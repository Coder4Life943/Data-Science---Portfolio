# SQL Database Design – Vaccine Clinic (2024)

## Module
Database Systems & Security

## Grade
2.1

## Project Overview
Designed and implemented a relational SQL database for a vaccine clinic business scenario.

The goal was to create a structured system to manage:

- patients
- staff
- appointments
- vaccines
- inventory

## Skills Demonstrated
SQL  
Database design  
Entity Relationship Diagrams (ERD)  
Data normalisation  
Primary and foreign keys  
SQL queries  
INNER JOIN operations  
Data security principles  

## What I built
I created a database system that allows a clinic to:

- store patient records securely
- schedule vaccination appointments
- track vaccine stock levels
- manage staff roles and responsibilities
- analyse vaccination data using SQL queries

## Technical Details
The database includes multiple related tables:

- Patients
- Staff
- Vaccines
- Appointments
- Inventory
- Doses Administered

The system uses:

- Primary Keys
- Foreign Keys
- Normalisation (1NF, 2NF, 3NF)
- SQL CREATE statements
- SQL INSERT statements
- SQL SELECT queries
- JOIN queries

## Security Considerations
The project also discusses:

- data privacy
- role-based access control
- encryption
- ethical handling of patient information
